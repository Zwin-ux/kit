# Instructions for Coding Agents (especially Grok Build)

You are working on **Kit** — the terminal-native, pixel-art Agent Skills platform.

## Project Goals (do not lose these)
- Portable skills that work across Claude Code, Grok Build, Codex, etc.
- 100% terminal experience (pixel-art TUI is the product, no web app)
- Free accounts from day one (Railway backend) for mass adoption
- High-quality pixel art personality (mascot is a baby fox named Kit holding a wrench)
- Strict validation, offline-first local engine, delightful UX

## Current Priority
Phase 0 → Phase 1: Foundation + Local Skills Engine.

Focus on clean architecture, strict skill schema, and a solid TypeScript core before heavy TUI work.

## Key Documents
- README.md — vision
- ARCHITECTURE.md — system design
- ROADMAP.md — phases
- docs/PIXEL_ART.md — visual direction

## Style Preferences
- Prefer clear, modern TypeScript
- Strong types and validation
- Small, focused modules
- Good error messages
- No web frontend code

## Mascot
When relevant, the mascot is a cute but competent pixel-art fox (kit) with a wrench. Keep the personality consistent: helpful, slightly playful, never corporate or childish.

## Working Style
- Make focused, reviewable changes
- Update docs when you change architecture or user-facing behavior
- Prefer boring, reliable solutions over clever ones
- Ask for clarification if a decision would lock us into a major direction

Grok Build is the preferred coding agent for this repository.
