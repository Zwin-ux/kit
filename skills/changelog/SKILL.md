# Instructions

1. Collect the user-facing changes since the last release note.
2. Group entries under Added, Changed, Fixed, or Removed.
3. Write for users, not for commit history.
4. Mention breaking changes first and say how to migrate.
5. Keep each bullet one idea long.
6. Update `CHANGELOG.md` or the project’s release notes file.
